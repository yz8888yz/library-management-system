**背景**: 传统图书馆管理依赖人工操作，效率低下，容易出错。需要开发一个数字化、自动化的图书管理系统。

## 2. 系统需求规格说明

### 2.1 功能需求

#### 2.1.1 用户管理模块

**用例**: UC001 - 用户注册
- **参与者**: 新用户
- **前置条件**: 用户未注册
- **基本流程**:
  1. 用户进入注册页面
  2. 输入用户名、邮箱、密码等信息
  3. 系统验证输入数据
  4. 系统检查用户名/邮箱唯一性
  5. 系统创建用户账户
  6. 返回注册成功信息
- **后置条件**: 用户账户创建完成

**用例**: UC002 - 用户登录
- **参与者**: 已注册用户
- **基本流程**:
  1. 用户输入用户名和密码
  2. 系统验证用户身份
  3. 验证账户状态（是否激活）
  4. 系统生成会话
  5. 返回登录成功
- **异常流程**: 密码错误或用户不存在，提示错误信息

**用例**: UC003 - 管理个人信息
- **参与者**: 已登录用户
- **功能**:
  - 查看个人信息
  - 编辑真实姓名、电话等
  - 修改密码
  - 查看借阅历史

**用例**: UC004 - 用户管理（仅管理员）
- **参与者**: 系统管理员
- **功能**:
  - 查看所有用户
  - 禁用/启用用户账户
  - 修改用户角色
  - 删除用户

#### 2.1.2 图书管理模块

**用例**: UC005 - 浏览图书
- **参与者**: 所有用户
- **功能**:
  - 查看图书列表（分页）
  - 按分类筛选
  - 查看图书详情（ISBN、作者、出版社等）
  - 显示可用数量

**用例**: UC006 - 搜索图书
- **参与者**: 所有用户
- **功能**:
  - 按书名搜索
  - 按作者搜索
  - 按ISBN搜索
  - 搜索结果分页显示

**用例**: UC007 - 管理图书（仅图书管理员）
- **参与者**: 图书管理员、管理员
- **功能**:
  - 添加新图书
  - 编辑图书信息
  - 删除图书
  - 管理图书分类
  - 更新库存数量

#### 2.1.3 借阅管理模块

**用例**: UC008 - 借阅图书
- **参与者**: 普通用户
- **前置条件**: 用户已登录，图书可用
- **基本流程**:
  1. 用户选择要借阅的图书
  2. 系统检查图书是否可用
  3. 系统检查用户是否有逾期未还的图书
  4. 创建借阅记录
  5. 更新图书库存
  6. 返回借阅成功
- **后置条件**: 借阅记录创建，图书可用数量-1

**用例**: UC009 - 归还图书
- **参与者**: 普通用户
- **基本流程**:
  1. 用户选择要归还的图书
  2. 系统查找对应的借阅记录
  3. 系统计算是否逾期
  4. 若逾期，计算罚款
  5. 更新借阅记录状态
  6. 更新图书库存
  7. 返回归还成功

**用例**: UC010 - 续借图书
- **参与者**: 普通用户
- **约束条件**: 最多续借3次，未逾期
- **功能**:
  - 延期30天
  - 记录续借次数

**用例**: UC011 - 查看借阅记录
- **参与者**: 普通用户、管理员
- **功能**:
  - 查看个人借阅历史
  - 按状态筛选（已借出、已归还、逾期、遗失）
  - 查看逾期罚款信息

**用例**: UC012 - 管理逾期记录（仅管理员）
- **参与者**: 图书管理员、管理员
- **功能**:
  - 查看逾期图书列表
  - 通知用户
  - 记录罚款
  - 处理遗失图书

### 2.2 非功能需求

- 用户密码采用加密存储

- 关键操作记录审计日志

- 系统可用性 > 99%
- 数据备份周期 ≤ 24 小时
- 故障恢复时间 < 1 小时

#### 2.2.5 可扩展性需求

```
          |
```

### 3.2 1层DFD - 主要流程
```
用户 ---> [1.0 用户管理] --> 用户数据
       --> [2.0 图书管理] --> 图书数据
```
