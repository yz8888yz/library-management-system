## 1. 系统架构设计

### 1.1 整体架构

采用 **MVC（Model-View-Controller）分层架构** + **服务层架构**

```
┌─────────────────────────────────────────┐
│          表现层 (View)                   │
│    HTML Templates + JavaScript           │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│          控制层 (Controller)              │
│    Flask Blueprints & Routes             │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│          业务层 (Service)                │
│    User/Book/Borrow Service              │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│          数据层 (Model)                  │
│    SQLAlchemy ORM Models                 │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│          数据库层                        │
│    MySQL Database                        │
└─────────────────────────────────────────┘
```

### 1.2 分层说明

**表现层 (View)**
- 负责用户界面展示
- HTML模板渲染
- 前端交互逻辑

**控制层 (Controller)**
- 接收用户请求
- 调用业务逻辑
- 返回响应结果
- 路由分发

**业务层 (Service)**
- 实现核心业务逻辑
- 数据验证和处理
- 跨模块协调

**数据层 (Model)**
- 定义数据结构
- ORM映射
- 数据库关系

**数据库层**
- 数据持久化
- 事务管理

### 2.3 索引设计

```sql
-- 性能优化索引
CREATE INDEX idx_username ON users(username);
CREATE INDEX idx_email ON users(email);
CREATE INDEX idx_isbn ON books(isbn);
CREATE INDEX idx_title ON books(title);
CREATE INDEX idx_author ON books(author);
CREATE INDEX idx_borrow_user_status ON borrow_records(user_id, status);
CREATE INDEX idx_borrow_due_date ON borrow_records(due_date);
```

## 8. 安全设计

### 8.1 身份认证
- 密码使用 Werkzeug 加密存储
- 支持 Flask-Login 会话管理
- 登录失败次数限制（可扩展）

### 8.2 访问控制
- 基于角色的访问控制 (RBAC)
- 三个角色：admin, librarian, user
- 装饰器验证权限

### 8.3 数据安全
- SQL 参数化查询（SQLAlchemy ORM）
- XSS 防护
- CSRF 防护（Flask-WTF）

### 8.4 审计日志
- 记录关键操作
- 记录用户登录信息
